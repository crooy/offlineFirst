What is a build tool

A build tool basically makes sure that your project ends up as an executable file (.jar / .war).

It helps you automate repetitive tasks such as:

  • ♥ compile source code into binary
  • ♥ package the binary code into executable files
  • ♥ running tests
  • ♥ download and manage dependencies
  • ♥ deploy your project onto production system

Wait. Dependencies?

Dependencies are the libraries, plugins, tools, frameworks that your web app is using (depending on).

These dependencies have dependencies of their own.

Sometimes they will depend on different and incompatible versions of the same library.

Fixing one dependency will then break another.

This is professionally referred to as

Dependency Hell!!

You do not want to go there.

Maven is not alone

And you also have Grunt, Gulp, SBT, Webpack, Rake, Cake🎂

How does it work

The most important file is the POM (Project Object Model).

POM contains the structure of the project, and it is up to Maven to build it, based on this info.

A POM file does not contain code, it is an XML representation of a Maven project.

I know it is vague. But understanding it all would take the magic out.

When to use build tools

It is always reccommended to use a build tool.

Advantages of build tools in general are:

  • Reproducable build: not depending on IDE or computer
  • Saves a lot of time compared to manual build
  • Less human typing = less chance of mistakes
  • Runs your unit tests for you
  • Automatically deploy new builds - always up to date
  • Autmatically check if you did not break anything

Why Maven?

Maven is specifically known for dependency management

and project management options.

  • Uniformity: all Maven projects work the same way
  • Dependencies are managed for you
  • Document structure is forced / standardised
  • Provides project information such as versions etc
  • Maven Archetypes: project templates that others can use

Why not?

Maven is not very flexible.


The structure convention is forced.
